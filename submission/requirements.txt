# AI Inference Challenge — pinned dependencies
#
# IMPORTANT: `pip install -r requirements.txt` alone is NOT safe on Blackwell.
# Use the accompanying setup.sh, which has a two-path install strategy:
#
#   Fast path (platform has torch pre-installed with cu128 Blackwell wheels):
#     - vllm installed with `pip install --no-deps` so pip cannot replace torch
#       with PyPI's default cu126 wheel (which has no sm_120 GPU kernels).
#     - All of vllm's transitive deps (transformers, tokenizers, xformers, ray,
#       triton, compressed-tensors, …) are assumed already present since the
#       platform ships vLLM itself.
#
#   Fallback path (greenfield: no torch installed):
#     - Bootstrap torch 2.7.0 + torchaudio + torchvision from the pytorch.org
#       cu128 index URL (NOT PyPI).
#     - Then `pip install vllm==0.9.2` normally. PEP 440 local-version matching
#       means the freshly-installed torch 2.7.0+cu128 satisfies vLLM's hard
#       `torch==2.7.0` requirement without being replaced; xformers/ray/etc.
#       come from PyPI.
#
# vllm 0.9.2 chosen because:
#   - has the new `--no-enable-log-requests` flag (required on platform)
#   - has `--speculative-config` JSON (replaces removed --speculative-model)
#   - matches platform's declared 0.9+ series (first-upload CLI-error evidence)
vllm==0.9.2
